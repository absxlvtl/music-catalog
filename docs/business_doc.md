# Music Catalog

## Опис системи
**Music Catalog** — це застосунок для збереження та управління музичними треками.  
Користувач може переглядати список пісень, додавати нові треки до бази даних і взаємодіяти з ними через зручний клієнтський інтерфейс.  
Система складається з клієнта (WPF), який надсилає запити до Node.js сервера, що зберігає дані у базі SQLite.

---

## Користувачі (ролі)
- **Звичайний користувач** — може переглядати список пісень і додавати нові.
- **Адміністратор (у майбутньому)** — зможе редагувати й видаляти треки.

---

## Основні сценарії використання
1. Користувач відкриває застосунок і натискає кнопку “Завантажити треки”.
2. Система надсилає запит `GET /tracks` до сервера.
3. Сервер повертає JSON із переліком треків.
4. Користувач бачить список на екрані.
5. За потреби користувач додає нову пісню через форму, що надсилає `POST /tracks`.

---

## Функціональні вимоги
- Система має повертати список усіх пісень з бази даних.
- Можливість додавання нового треку.
- Збереження даних у SQLite.
- Обробка запитів через REST API (GET, POST).

---

## Нефункціональні вимоги
- **Швидкодія:** відповіді API повинні приходити не довше 1 секунди.
- **Безпека:** у майбутньому планується авторизація користувачів.
- **Зручність:** інтерфейс має бути інтуїтивно зрозумілим.
- **Портативність:** система повинна запускатися локально без складних налаштувань.

---

## Обмеження
- Немає онлайн-синхронізації між пристроями.
- Система не підтримує відтворення музики — лише збереження інформації.
- Авторизація поки що відсутня (буде реалізована в наступних версіях).
